# 0005: Document.replaceString carve-out for in-quotes content edits in dependsOn

**Status:** Accepted
**Date:** 2026-05-04

`ReplaceWithNearMatchFix` (in `DependsOnQuickFixes.kt`) rewrites the content of a `JsonStringLiteral` inside a `dependsOn` array (e.g. replacing `"buld"` with `"build"`). The canonical PSI-driven idiom for content edits to a `JsonStringLiteral` is `ElementManipulators.handleContentChange(element, replacement)`, as documented in `research/json-psi.md` and used by the solar-form-helper reference plugin. We attempted to port `ReplaceWithNearMatchFix` to that idiom on 2026-05-04. It compiled cleanly and produced a string-correct result, but the resulting file diverged from the test fixture: `handleContentChange` invokes the bundled JSON code-style reformatter as part of the PSI replacement, and the reformatter expands a single-line `dependsOn` array such as `"dependsOn": ["build"]` into the multi-line form `"dependsOn": [\n      "build"\n    ]`. The behavior is deterministic and reproducible against the existing fixture `inspections/dependsOnUndefinedTask/unknownTaskWithNearMatch/turbo.json`. The expansion is a real user-visible change, not a whitespace nit: it rewrites the user's hand-formatted single-line array on a quick fix that they expected to swap one identifier and leave the rest of the file alone.

We are keeping `Document.replaceString` as the implementation for `ReplaceWithNearMatchFix`. This is a deliberate carve-out from ADR 0004's "use canonical PSI helpers, not document-level edits" guidance, scoped to the case of in-quotes content edits where the canonical PSI helper triggers a reformatter and the reformatter materially changes layout the user did not ask to change. The `Document.replaceString` path replaces only the literal's text range with the new quoted content and preserves the surrounding line layout. If a future platform release changes `handleContentChange`'s reformatter behavior (e.g. by gating it on a code-style flag we can disable for this call site), or if we add a cross-cutting "no reformat on quick fix" wrapper, this carve-out should be revisited and `ReplaceWithNearMatchFix` ported. The carve-out applies only to literal-content edits inside a `dependsOn` array as of 2026-05-04; structural edits (adding/removing/replacing properties or values) continue to follow ADR 0004's `JsonPsiUtil.addProperty`, `JsonElementGenerator.createValue<T>`, and `WriteCommandAction` patterns.
