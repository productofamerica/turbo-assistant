# v0.1 Specification

## Goal

A WebStorm/IntelliJ plugin that surfaces cross-file errors in `turbo.json` that the JSON schema can't catch, plus navigation between `turbo.json` task definitions and `package.json` scripts. Target IDEs: WebStorm and IntelliJ Ultimate, plus other JS-capable JetBrains IDEs.

## In scope

### Inspection A — unresolved `dependsOn` reference

A `dependsOn` entry references a task that no package's `turbo.json` or `package.json` defines. Reports on the offending string element. Quick fix: remove the entry, or — when a near match exists — replace it with the closest known task name.

### Inspection B — unresolved package-qualified task

A package-qualified task reference (e.g. `"web#build"`) names a package that doesn't exist in the workspace. Reports on the package portion of the string. Quick fix: replace with the closest matching workspace package name.

### Inspection C — malformed `extends`

A package-level `turbo.json` (anywhere except the workspace root) has an `extends` field whose value is not exactly `["//"]`. Reports on the value. Quick fix: replace the value with `["//"]`.

### PsiReference — task → task definition

Cmd-click on a task name inside `dependsOn` jumps to that task's definition under `tasks.*` in the same `turbo.json`, or in the appropriate package-level `turbo.json` for qualified references.

### PsiReference — script → turbo task

Cmd-click on a script name in `package.json` jumps to the matching `turbo.json` task entry, if one exists. Silent (no resolution, no error) when no matching task is defined.

### JSONC support

All three inspections, both PsiReferences, and every quick fix apply equally to `turbo.jsonc`.

## Out of scope for v0.1

- Env var detection (v0.3).
- Gutter run icons and Run Configuration generation (v0.2).
- Package graph tool window (v1.0).
- Run summary viewer (backlog).
- Boundaries integration (v0.4).
- Codemod runner UI (backlog).
- Microfrontends integration (backlog).
- Anything that shells out to the `turbo` CLI.

## Acceptance criteria

Each inspection has a fixture test using `BasePlatformTestCase` that loads a violating input from `src/test/testData/` and asserts:

1. The expected `ProblemDescriptor` is reported (correct range, correct severity, correct message key).
2. The quick fix, when applied, transforms the file into the expected output fixture.

Each PsiReference has a fixture test using `BasePlatformTestCase` that places the caret on the reference site and asserts the resolved target PSI element matches the expected location in a sibling fixture file.
